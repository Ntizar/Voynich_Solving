The Approach

While most Voynich research treats the text as one-dimensional prose, we treat it as what it structurally is: a two-dimensional database. Each line is a record. Each column position has a function. Suffixes encode categories. This is not a cipher over natural language -- it is a structured notation system for pharmaceutical recipes.

El Enfoque

Mientras la mayoria de la investigacion Voynich trata el texto como prosa unidimensional, nosotros lo tratamos como lo que estructuralmente es: una base de datos bidimensional. Cada linea es un registro. Cada posicion de columna tiene una funcion. Los sufijos codifican categorias. No es un cifrado sobre lenguaje natural -- es un sistema de notacion estructurado para recetas farmaceuticas.

Descubrimientos Clave

#1

Functional Suffix Channel

Canal de Sufijos Funcionales

Voynich words decompose as STEM + FINAL_ATOM. The suffix carries grammatical function, not meaning. H(final|stem) = 1.246 bits vs null 2.096 bits (Z = -210). 82.2% of stems take only one suffix. Dominant endings: A2, B2, C1, G1 (79.1%).
Las palabras Voynich se descomponen como STEM + ATOMO_FINAL. El sufijo porta funcion gramatical, no significado. H(final|stem) = 1.246 bits vs null 2.096 bits (Z = -210). 82.2% de stems solo admiten un sufijo. Terminaciones dominantes: A2, B2, C1, G1 (79.1%).
#2

2D Vertical Alignment

Alineacion Vertical 2D

NOVEL FINDING. Functional suffixes align vertically across consecutive lines at 27% probability vs 8% in Latin/Spanish. This proves the text was written as a table, not prose.
HALLAZGO NUEVO. Los sufijos funcionales se alinean verticalmente entre lineas consecutivas con 27% de probabilidad vs 8% en Latin/Espanol. Esto prueba que el texto fue escrito como tabla, no como prosa.
#3

Cross-Section Foreign Keys

Claves Foraneas entre Secciones

46.9% of exclusive botany stems reappear in pharmacy/recipe sections with suffix changes (C1 to B2/B3). This proves a relational database structure linking plant catalog to recipe formulas.
46.9% de stems exclusivos de botanica reaparecen en secciones de farmacia/recetas con cambios de sufijo (C1 a B2/B3). Esto prueba una estructura de base de datos relacional conectando el catalogo de plantas con formulas de recetas.
#4

Recipe Matching v7 F1 UNDER REVIEW

Matching de Recetas v7 F1 EN REVISION

35 out of 48 recipe-section folios match real historical recipes at F1 >= 80%. Mean F1 = 81.9%. However, validation (Session 15) revealed the F1 metric is non-discriminative: a majority-recipe baseline achieves 100% F1, and a most-common-ingredients baseline achieves 90.8%. The 81.9% claim requires alternative metrics to confirm. Structural discoveries remain unaffected.

Exploration Paths

Where we go from here.

Vias de Exploracion

A donde vamos desde aqui.

Priority 1 -- CRITICAL

Fix the F1 Metric

The current F1 metric is non-discriminative: a majority-recipe baseline scores 100%, and most-common-ingredients scores 90.8%. We need discriminative F1 (excluding ingredients in >80% of recipes), ranking accuracy (does the system rank the correct recipe higher?), and exclusion accuracy (does the system correctly predict ingredient ABSENCE?). This is the single most important next step.

Arreglar la Metrica F1

La metrica F1 actual no es discriminativa: una baseline de receta mayoritaria logra 100%, e ingredientes mas comunes logra 90.8%. Necesitamos F1 discriminativo (excluyendo ingredientes en >80% de recetas), precision de ranking (el sistema rankea la receta correcta mas alto?), y precision de exclusion (el sistema predice correctamente la AUSENCIA de ingredientes?). Este es el paso siguiente mas importante.

Priority 2

Expand Recipe Database to 100+

Our 50-recipe database needs expansion. More recipes from Circa Instans, Thesaurus Pauperum, and Lumen Apothecariorum will increase discrimination power, break remaining deadlocks, and provide better baselines for the new metrics.

Expandir Base de Recetas a 100+

Nuestra base de 50 recetas necesita expansion. Mas recetas del Circa Instans, Thesaurus Pauperum y Lumen Apothecariorum aumentaran el poder discriminante, romperan deadlocks restantes y proporcionaran mejores baselines para las nuevas metricas.

Priority 3

Complete Validation Phases 5-10

Remaining phases: transliteration round-trip test, contradiction engine, bootstrap/ablation analysis, FDR correction, automated semaphore report. These will provide comprehensive confidence calibration for all identifications.

Completar Fases de Validacion 5-10

Fases restantes: prueba de ida y vuelta de transliteracion, motor de contradicciones, analisis bootstrap/ablacion, correccion FDR, reporte semaforo automatizado. Proporcionaran calibracion de confianza integral para todas las identificaciones.

Priority 4

Break Remaining Deadlocks

Galanga/Cubeba/Nux moschata triple deadlock is unbreakable with 50 recipes (47 TIED). Opium/Castoreum is partially broken (9 Castoreum stems confirmed). External sources with simpler formulas, morphological analysis, or botanical cross-reference may help.

Romper Deadlocks Restantes

El deadlock triple Galanga/Cubeba/Nux moschata es irrompible con 50 recetas (47 EMPATE). Opio/Castoreum parcialmente roto (9 stems de Castoreum confirmados). Fuentes externas con formulas mas simples, analisis morfologico o referencia cruzada botanica pueden ayudar.

Priority 5

Positional Grammar & Botany Section

Build a formal grammar model from column positions: [ENTITY]→[PROPERTY]→[ACTION LIST]. Cross-correlate 217 botany stems with known medieval simples. Zingiber and Crocus identifications can be verified against botanical illustrations.

Gramatica Posicional y Seccion Botanica

Construir modelo gramatical formal de posiciones: [ENTIDAD]→[PROPIEDAD]→[LISTA ACCIONES]. Correlacionar 217 stems botanicos con simples medievales conocidos. Las identificaciones de Zingiber y Crocus pueden verificarse contra ilustraciones botanicas.

Priority 6

Statistical Publication

Formalize the suffix channel, 2D alignment, and foreign key discoveries as a statistical paper. The 27% vertical alignment rate (vs 8% null) and Z=-210 suffix entropy are publication-grade results that stand independent of any decipherment claims or the broken F1 metric.

Publicacion Estadistica

Formalizar los descubrimientos del canal de sufijos, alineacion 2D y claves foraneas como articulo estadistico. La tasa de alineacion vertical del 27% (vs 8% nulo) y Z=-210 de entropia de sufijos son resultados publicables que son independientes de las afirmaciones de desciframiento o la metrica F1 rota.
